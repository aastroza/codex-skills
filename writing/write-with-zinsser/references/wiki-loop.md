# Writing Wiki Loop

This skill draws from the agent-starter-wiki idea: a writing guide should grow from real sessions.

Use this loop after a productive writing or editing session:

1. Identify what the agent got wrong or what the human corrected.
2. Turn the correction into a reusable rule.
3. Add the rule to the smallest relevant reference file.
4. Include a before/after example when the rule depends on taste.
5. Keep the rule specific. "Avoid bad tone" is weak. "Do not open with an apology before the argument" is useful.

Good entries capture judgment:

- why a draft structure felt false
- why one verb fit better than another
- when a technical detail earned its place
- which examples served the argument
- which endings sounded too neat

Do not add session logs, broad manifestos, or private context that the next writing task will not need.
